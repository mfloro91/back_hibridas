Documentación de la API

Introducción

Esta aplicación ha sido creada para ayudar a los hoteles a administrar los servicios brindados a los huespedes.

Comprando nuestra app, los hoteles podrán customizar la aplicación, cargando sus datos como así mismo información de los distintos servicios disponibles para los huéspedes.

Una vez customizada la app y dados de alta los servicios, los huespedes podrán empezar a hacer sus reservas, mejorando su estadía. Así mismo, los hoteles podrán gestionar la experiencia de los usuarios, mejorando y organizando la atención al cliente.

Nuestra API

Esta documentación te ayudará a familiarizarte con los recursos de la API de hoteles y te mostrará cómo realizar diferentes consultas.

Antes de comenzar, es super importante que sepas que de acuerdo al rol que tengas, nuestra aplicación te ofrecerá distintos accesos a la información.

Hotels

Hotel Schema

Key Type Description
id int El ID del hotel (es automático).
name string El nombre del hotel.
logo string El logo que utiliza el hotel.
description string Una breve descripción o slogan.
languages array(string) Lenguajes a los que deberá traducirse la interfaz.
country string País donde se ubica el hotel.
city string Ciudad o localidad en la que se ubica el hotel.

Get all hotels: listado de hoteles

Endpoint: Podrás acceder a la funcionalidad usando el endpoint GET http://localhost:3000/hotels

Funcionalidad: Es un listado de todos los hoteles, solo disponible para superadmin. Esto permite al equipo conocer todos los clientes u hoteles que han comprado la app. 


                    
                        [
                            {
                                "_id": "6649189a5c8e4a001fa3a001",
                                "name": "Hotel Oasis",
                                "logo": "https://example.com/logos/oasislogo.png",
                                "description": "Un refugio urbano con spa, piscina y gastronomía internacional.",
                                "languages": [
                                    "es",
                                    "en",
                                    "fr"
                                ],
                                "country": "Argentina",
                                "city": "Buenos Aires"
                            }
                        ]

Filter Hotels: Búsqueda por ubicación

Funcionalidad: Los superadmin podrán buscar los hoteles según País o Ciudad. Ingresando en el query params un campo de texto, arrojará todos los resultados que incluyan dicho texto.

Create hotel: creación nuevo hotel

Endpoint: Podrás acceder a la funcionalidad usando el endpoint POST http://localhost:3000/hotels

Funcionalidad: Los superadmin podrán crear un nuevo hotel completando ciertos campos, mencionados a continuación. Los campos que se listan con un * son considerados obligatorios.

Campos a ingresar en el body:

  • Name*
  • Logo
  • Languages (array)
  • Country*
  • City*

                    
                        POST localhost:3000/hotels
                        
                        // Body JSON:

                       {
                            "name": "Los arcos",
                            "logo": "https://example.com/logos/arcos.png",
                            "description": "Edificio de departamentos temporarios.",
                            "languages": [
                                "es",
                                "en"
                            ],
                            "country": "Uruguay",
                            "city": "Montevideo"
                        }

Edit hotel: editando hotel existente

Endpoint: Podrás acceder a la funcionalidad usando el endpoint PUT http://localhost:3000/hotels/:id

Funcionalidad: Una vez que el superadmin crea el hotel, el administrador del mismo tendrá permisos para editar información disponible.


                    
                        PUT localhost:3000/hotels/6649189a5c8e4a001fa3a002

                        // Body JSON:

                        {
                            "languages": [
                                "es"
                            ]
                        }

Delete hotel: eliminar hotel existente

Endpoint: Podrás acceder a la funcionalidad usando el endpoint DELETE http://localhost:3000/hotels/:id

Funcionalidad: Solo los superadmin podrán eliminar clientes de su cartera. 


                    
                        DELETE localhost:3000/hotels/682274971462c2e1cd694bc9

Create service: crear nuevo servicio

Endpoint: Podrás acceder a la funcionalidad usando el endpoint POST http://localhost:3000/services

Funcionalidad: Los superadmin, admin o miembros del staff podrán crear un nuevo servicio, así mismo como editarlo o eliminarlo.

Campos a ingresar en el body:

  • Title*
  • Description
  • Available Hours*
  • Hotel_id (se ingresa automáticamente desde los datos del login del usuario. Esto permite enlazar el servicio con el hotel donde se brinda. Las personas solo podrán ingresar servicios para el hotel que administran)

                    
                        POST localhost:3000/services
                        
                        // Body JSON:

                       {
                            "title": "Cuidado de niños",
                            "description": "Espacios personalizados, para que los peques se diviertan y los adultos puedan desconectar.",
                            "availableHours": "10:00 - 20:00"
                        }

Key Type Description
id int El ID del pedido (es automático).
hotel_id ObjectId El ID del hotel donde se realiza el pedido.
service_id ObjectId El ID del servicio sobre el cual se solicita un pedido.

note string Se pueden agregar notas al pedido o reserva.

Filter & Order orders: visualizar pedidos y filtrarlos por estado o habitación

Endpoint: Podrás acceder a la funcionalidad usando el endpoint GET http://localhost:3000/orders

Funcionalidad: Es un listado de todos los pedidos realizados por los huespedes y solo pueden acceder a dicha información los administradores o miembros del staff. Tiene un filtro por hotel, que se toma de los datos del usuario al hacer login. Posteriormente los admins podrán indicar en el query el número de la habitación a filtrar (room_number) o el estado del pedido (status). Por último, la información se ordenará de manera ascendente, mostrando los pedidos más antiguos o prioritarios primero.

Sin aplicar ningún query adicional, se aplica un filtro automático de acuerdo al hotel asociado a mi perfil:

                                    "_id": "66491aa75c8e4a001fa3b002",

                                    "availableHours": "08:00 - 23:00"

Create order: crea un nuevo pedido

Endpoint: Podrás acceder a la funcionalidad usando el endpoint POST http://localhost:3000/orders

Funcionalidad: Cualquier usuario final podrá crear un pedido.

Campos a ingresar en el body:

  • Service_id* (permite establecer una relación entre el pedido y el tipo de servicio solicitado)
  • Room_number*
  • Note
  • Status (no es necesario que el usuario cargue esta info, por defecto el status siempre será "pendiente" al crear una nueva orden)
  • Hotel_id (se completa automáticamente desde el perfil del usuario. De este modo, solo podrá hacer reservas dentro del hotel correcto).

                    
                        POST localhost:3000/orders
                        
                        // Body JSON:

                        {
                            "hotel_id": "6649189a5c8e4a001fa3a001",
                            "service_id": "66491aa75c8e4a001fa3b001",
                            "room_number": 5,
                            "note": "Tengo presión baja"
                        }

Edit order: edita el estado de un pedido

Endpoint: Podrás acceder a la funcionalidad usando el endpoint PATCH http://localhost:3000/orders/:id

Funcionalidad: Podrás editar el estado del pedido si estás logueado como superadmin, admin o staff. En el body del request, deberás indicar el nuevo estado, el cual puede ser "pendiente", "completado", "en proceso". 


                    
                        PATCH localhost:3000/orders/682295b52350c322db70f45e

                        // Body JSON:

                        {
                            "status": "Completado"
                        }

Funcionalidad: Podrás eliminar un pedido existente logueandote como admin o staff.

hotel_id ObjectId El ID del hotel donde se hospeda el huesped (re-pensar cómo se asigna).

Create new user: crea nuevo usuario

Endpoint: Podrás acceder a la funcionalidad usando el endpoint POST http://localhost:3000/users

Funcionalidad: Permite crear nuevos usuarios. Por defecto el rol al crear un nuevo usuario siemrpe será "user". Si sos administrador o staff de un hotel, deberás esperar que un superadmin te cambie los permisos.

Campos a ingresar en el body:

  • Hotel_id* (permite establecer una relación entre el usuario y el hotel donde se hospeda/trabaja - a futuro debería poder completarse automáticamente - repensar login de usuarios finales)
  • Name*
  • userName
  • Email*
  • Role (no es necesario que el usuario cargue esta info, por defecto el rol siempre será "user" al crear una nuevo usuario)
  • Password*

                    
                        
                        POST localhost:3000/users
                        
                        // Body JSON:

                            {
                                "hotel_id": "6649189a5c8e4a001fa3a002",
                                "name": "Eduardo",
                                "userName": "edu123",
                                "email": "edu@gmail.com",
                                "password": "123"
                            }

Edit user role: edita el rol del usuario

Endpoint: Podrás acceder a la funcionalidad usando el endpoint PATCH http://localhost:3000/users/:id

Funcionalidad: Por seguridad, siempre que se cree un nuevo usuario por defecto siempre tendrá el rol "user". Sin embargo, los superadmin tienen la posibilidad de editar el rol del usuario, para asignar admins o miembros del staff de un hotel. En el body deberás ingresar el nuevo "role", el cual puede ser "admin", "staff" o "superadmin". 


                    
                        PATCH localhost:3000/users/6822995e2350c322db70f479

                        // Body JSON:
                            {
                                "role": "admin"
                            }
